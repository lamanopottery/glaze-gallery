# Glaze Gallery

A password-protected gallery for browsing photos of two-glaze combinations. The same codebase
supports multiple pottery studios.

The repository has two independent parts:

- `src/` — an Astro/TypeScript frontend deployed to Cloudflare Pages.
- `glaze_gallery/` — a Python pipeline that reads Google Sheets and Drive, processes images, and
  publishes the resulting images and JSON manifests to object storage.

The frontend only consumes the published manifests and images; it does not access Google APIs.

## Local development

Requirements: Node 22, Yarn 4, Python 3.11+, and Poetry.

```sh
yarn install
poetry install
cp .env.example .env
cp .dev.vars.example .dev.vars
yarn dev
```

Fill in `.env` with the pipeline settings and the Astro settings used by `yarn dev` and
`yarn build`. `.dev.vars` mirrors the Astro settings for the local Cloudflare/Wrangler runtime;
keep every value it contains in sync with `.env`. Ask a maintainer for the real data sources and
credentials. Studio names, contact details, app and static-data URLs, and branding are configured
in `src/utils/studio.ts`. Set `GLAZE_GALLERY_STUDIO` to `lamanopottery` or `mudmatters` in both env
files to select the studio.

## Common commands

```sh
yarn dev                         # Download manifests and start Astro
yarn build                       # Download manifests, typecheck, and build
yarn preview                     # Preview the build in Wrangler
yarn lint                        # Typecheck and check formatting
poetry run mypy glaze_gallery    # Typecheck the Python pipeline
poetry run download_images       # Rebuild images and manifests from Google data
```

`src/data/` is generated by `download_data` whenever the frontend starts or builds. Do not edit it
by hand. The heavier `download_images` pipeline requires Google OAuth credentials and is intended
to be run manually when source data changes.
